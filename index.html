내가 생각하는 사용자경험 - User Experience

UX의 정의에 대해서 알아보고, 다양한 UX의 사례들, 특히나 성공적인 UX라 불리는 사례들을 보면서
사용자 경험을 바탕으로 어떤 제품이나 서비스를 구축하는 것이 중요하다는걸 깨달을 수 있었다. 내가 생각하는 UX란, 단순히 예쁘게 보이는 것이 중요한 것이 아니라 그 모양이 어떻게 작용하느냐가 중요하다고 생각한다.
사용자가 쉽게 즐기며 재밌어서 자주 사용하는 것이 성공적인 UX가 아닐까 ?
사전적인 의미에서 UX란 사람의 여러 감각과 감정의 총합을 뜻하는데, 내가 생각하는 UX는 부정적 경험을 통해 느끼는
부정적인 감정을 최소화시키는 것이라는 생각이 든다.

1.'웹 퍼블리셔'를 선택한 이유 ?
전공은 메이크업을 기반으로 외국계 백화점 화장품 판매사원부터 미용관련 하이모에서 카운셀러등등 서비스업에 종사하였습니다. 제가 했던 직종이 활동할 수 있는 연령층이 낮다보니 연령 제한없는 평생직업군으로 직종변경을 고민하던중 개발쪽에 있는 지인의 권유로 시작하게 되었습니다.
2. 웹표준, 웹접근성, 시멘틱웹의 뜻과 차이점 ?
웹표준이란 똑같은 HTML문서를 다른 브라우저에서 열었을때 디자인은 조금씩 다를순 있어도 기능적으로는 같도록 만든 규칙입니다. 웹접근성이란 어떠한 환경에서도 장애인이나 비장애인까지 아울러 웹사이트를 이용할 수 있도록 하는 만드는것입니다. 시멘틱웹이란 개발자가 의도한 요소의 의미가 명확하게 작성하는 방법입니다. 외적인 디자인에 신경쓰지 말고, 컨텐츠의 국문학적 의미 즉, 역할에 맞는 태그를 사용하는 것입니다. HTML 작성시 모두다 지켜져야 할 내용이라고 생각합니다.
3. 리액트란 무엇이며 사용하는 이유 ?
리액트란 사용자 인터페이스를 만들기 위한 라이브러리이며 사용하는 이유는 리액트를 사용하면 재사용 가능한 UI 구성요소를 만들 수 있습니다. 컴포넌트를 레고처럼 다루어 레고를 조립하듯이 컴포넌트를 조립하여 유지보수 하기에 유리합니다. 리액트를 사용하면 페이지 전체를 렌더링하지 않고 렌더링이 필요한 부분만 렌더링 할 수 있습니다.
4. 리액트 훅이란 ?
리액트에서 제공하는 "마법 같은 함수"입니다. 클래스 형식의 리액트 컴포넌트에서만 할 수 있었던 state 관리나 라이프사이클등을 함수 형태 컴포넌트에서도 사용할 수 있게 만들어 주는 기능입니다. 리액트훅을 사용하면 좋은점은 코드가 간결해지고, 가독성이 좋아지며, 불필요한 것 같은 코드를 적을 필요가 없습니다.
5.가상돔 (Virtual DOM) 이란 ?
가상 DOM은 실제 DOM에서 처리하는 방식이 아닌 Virtual DOM과 메모리에서 미리 처리하고 저장한 후 실제 DOM과 동기화하는 프로그래밍 개념입니다.리액트에서는 리액트요소로 부터 가상돔이 만들어 지고 렌더링할때마다 가상돔을 만들어 이전의 가상돔과 비교합니다. 이는 실제돔의 변경사항을 최소화 하기 위한 과정입니다. 리액트는 렌더링의 성능을 위해 가상돔을 활용합니다.
